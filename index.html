IoT-проект: умная система полива растений BonchPlants

Описание проекта

WaterBot — это автономная система мониторинга и автоматического полива растений, построенная на базе микроконтроллера ESP32. Система собирает данные о влажности почвы и отправляет их на удаленный сервер через Интернет. Управление насосом может осуществляться как в автоматическом режиме, так и по команде с сервера.

Архитектура и компоненты

Общая схема работы: Датчик → ESP32 → Wi-Fi → ngrok → FastAPI Сервер → Команда → Реле → Насос.

Основные модули:

  • ESP32 DEVKIT V1: Микроконтроллер с интегрированным Wi-Fi для подключения к сети.
  • Датчик влажности: Аналоговый датчик, подключенный к GPIO34.
  • Релейный модуль: Используется для управления насосом (GPIO25).
  • Сервер FastAPI (Python): Хостится на VDS, выступает в качестве центра управления и хранения данных.
  • ngrok: Обеспечивает защищенный туннель для доступа ESP32 к локально работающему FastAPI.

Протокол обмена данными

Обмен данными происходит по протоколу HTTP/HTTPS в формате JSON.

1. Отправка статуса (Client → Server)

ESP32 отправляет текущие показания датчика каждые 10 секунд на сервер, используя метод POST.

Endpoint: POST /update

{ "soil": 950, "percent": 55 }

2. Получение команд (Client → Server)

ESP32 опрашивает сервер на предмет управляющих команд (например, включить насос или изменить режим) с помощью метода GET.

Endpoint: GET /command

// Пример ответа сервера для включения насоса и авторежима { "pump": "on", "auto": true }

(В прошивке реализован обход предупреждения ngrok с помощью заголовка ngrok-skip-browser-warning.)

Основные возможности

  • Автоматический полив: Насос включается, если влажность опускается ниже заданного порога (30%).
  • Удаленное управление: Возможность вручную включить/выключить насос или изменить режим через веб-интерфейс/API.
  • Калибровка: Возможность задать в NVS (энергонезависимая память) значения сухого и влажного состояния почвы через Serial Monitor.
  • Таймер безопасности: Автоматическое отключение насоса через 20 секунд (SAFETY_MAX_ON_MS) для предотвращения затопления.

Команды Serial Monitor (для отладки)

Даже без сервера, система позволяет управлять реле локально через Монитор порта:

pump on // Включить насос pump off // Выключить насос pulse 5 // Включить на 5 секунд cal dry // Калибровка сухого состояния status // Вывод текущих данных
